01
Was ist Authentifizierung?
Du willst dich im CyberCore-System anmelden. Das System verlangt Benutzername und Passwort. Welche Aussage beschreibt Authentifizierung am besten?
Das System entscheidet, welche Dateien du öffnen darfst.
Das System verschlüsselt dein Passwort beim Speichern.
Das System prüft, wer du bist — es verifiziert deine Identität.
Das System sichert deine Sitzung gegen Angriffe ab.
Das System verwaltet, wie lange du eingeloggt bleibst.
Hinweis anzeigen
„Authentisch" bedeutet echt, wirklich diese Person. Die Authentifizierung beantwortet die Frage: Bist du wirklich derjenige, der du behauptest zu sein?
02
Authentifizierung vs. Autorisierung
Du bist erfolgreich eingeloggt. Jetzt versuchst du die /schueler/noten-Seite zu öffnen — das System verweigert den Zugriff mit FORBIDDEN. Was passiert hier?
Die Authentifizierung schlägt fehl — dein Passwort ist abgelaufen.
Die Autorisierung verweigert den Zugriff — du hast keine Berechtigung für diese Seite.
Die Verbindung zum Server ist unterbrochen.
Das Betriebssystem blockiert den Zugriff aus Sicherheitsgründen.
Der Browser verhindert den Zugriff wegen fehlender Zertifikate.
Hinweis anzeigen
Du bist bereits eingeloggt — das System weiß wer du bist (Authentifizierung OK). Aber darf jeder eingeloggte Nutzer die Noten aller Schüler sehen? Das ist die Frage der Autorisierung.

04
Brute-Force-Angriff
Der Angreifer hat das Passwort durch wiederholtes Ausprobieren erraten — zweimal falsch, beim dritten Versuch richtig. Wie nennt man diese Methode?
Brute-Force-Angriff — systematisches Durchprobieren von Passwörtern.
Phishing — gefälschte E-Mails oder Webseiten um Passwörter zu stehlen.
SQL-Injection — Einschleusen von Code in Datenbankabfragen.
Man-in-the-Middle — Abfangen von Daten zwischen zwei Parteien.
Social Engineering — Menschen manipulieren um Informationen zu erhalten.
Hinweis anzeigen
„Brute" = rohe Gewalt. Keine Tricks, kein Täuschen — einfach so lange probieren bis es klappt. Stell dir vor jemand tippt stundenlang verschiedene Passwörter ein.

Welche der folgenden Eigenschaften machen ein Passwort sicher? Wähle alle richtigen Antworten aus.
Mindestens 8 Zeichen lang

Hinweis anzeigen
Denk daran: Was macht ein Passwort schwer zu erraten? Länge, Komplexität und Einzigartigkeit. Was macht es leicht zu erraten? Persönliche Infos und Muster.

Privilege Escalation
Nach dem Login erscheint im Log:
03:48:12PRIVILEGE ESCALATION — admin → root
Was bedeutet dieser Eintrag?
Der Angreifer hat ein neues Konto mit dem Namen „root" erstellt.
Der Angreifer hat das Passwort des Admin-Kontos geändert.
Der Angreifer hat sich vom Admin-Konto aus höhere Rechte verschafft — root darf alles.
Das System hat den Angreifer automatisch abgemeldet und als root neu eingeloggt.
Der Angreifer hat das Betriebssystem neu gestartet.
Hinweis anzeigen
„Privilege" = Berechtigung/Recht. „Escalation" = Steigerung. „root" ist auf Linux-Systemen der Superuser — der darf buchstäblich alles, keine Einschränkungen.

09
Zwei-Faktor-Authentifizierung (2FA)
Hätte 2FA den Angriff verhindert? Zuerst: Was ist das Grundprinzip der Zwei-Faktor-Authentifizierung?
Man braucht zwei verschiedene Passwörter nacheinander eingeben.
Das Passwort wird doppelt verschlüsselt übertragen.
Man muss sich von zwei verschiedenen Geräten gleichzeitig einloggen.
Zusätzlich zum Passwort braucht man einen zweiten Beweis — z.B. einen Code aufs Handy.
Das System schickt das Passwort zur Bestätigung per E-Mail zurück.
Hinweis anzeigen
2FA = zwei verschiedene Faktoren. Faktor 1: etwas das du weißt (Passwort). Faktor 2: etwas das du hast (Handy, Token) oder bist (Fingerabdruck). Zwei Passwörter wären immer noch nur ein Faktor.
