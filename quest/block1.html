// BLOCK 01 — ERSTZUGANG
Tatzeit: 08.09.2025 · 03:47 UTC · HLS Berlin Schulnetzwerk
// EINWEISUNG

Du hast Zugang zum System erhalten. Bevor du die Fallakte öffnen kannst, musst du beweisen dass du die Grundlagen der IT-Sicherheit kennst. Analysiere die Logs, beantworte die Fragen — und denk nach bevor du klickst.

FORTSCHRITT BLOCK 01 0 / 8 Aufgaben
01
Was ist Authentifizierung?
Das System verlangt einen Login bevor du die Fallakte öffnen kannst. Welche Aussage beschreibt Authentifizierung korrekt?
Das System prüft was du tun darfst — also deine Zugriffsrechte.
Das System prüft wer du bist — also deine Identität.
Das System speichert dein Passwort in einer Datenbank.
Das System verschlüsselt deine Verbindung zum Server.
Hinweis anzeigen
Authentifizierung kommt von „authentisch" — echt, wirklich diese Person. Denk an: Wer meldet sich hier an? Bin ich das wirklich?
02
Authentifizierung vs. Autorisierung
Du hast dich eingeloggt. Jetzt willst du die Notenliste aller Schüler öffnen — aber das System verweigert den Zugriff. Was passiert hier?
Die Authentifizierung schlägt fehl — dein Passwort ist falsch.
Das System hat einen technischen Fehler.
Die Autorisierung verweigert den Zugriff — du hast nicht die nötigen Rechte.
Die Verschlüsselung blockiert den Zugriff.
Hinweis anzeigen
Du bist eingeloggt — das System weiß wer du bist. Aber darf ein normaler Schüler die Noten aller anderen sehen? Das ist eine andere Frage.
03
Log-Analyse: Der erste Schritt
Hier sind die ersten Log-Einträge der Nacht. Was hat der Angreifer als erstes getan?
03:47:22LOGIN FAILED — user: admin — wrong_password
03:47:31LOGIN FAILED — user: admin — wrong_password
03:47:44LOGIN SUCCESS — user: admin

Er hat sofort Dateien gelöscht.
Er hat mehrfach versucht sich mit dem Admin-Konto einzuloggen.

Er hat eine neue Datei hochgeladen.
Hinweis anzeigen
Schau dir die Uhrzeit der ersten Einträge an. Was passiert zweimal kurz hintereinander, bevor der Login klappt?
04
Fachbegriff: Brute-Force
Der Angreifer hat das Passwort durch wiederholtes Ausprobieren erraten. Wie nennt man diese Angriffsmethode?
Brute-Force-Angriff — systematisches Durchprobieren vieler Passwörter.
Phishing — gefälschte E-Mails um Passwörter zu stehlen.
SQL-Injection — Angriff auf Datenbankabfragen.
Man-in-the-Middle — Abfangen von Daten zwischen zwei Parteien.
Hinweis anzeigen
„Brute" bedeutet auf Englisch „rohe Gewalt". Der Angreifer benutzt keine Tricks — er probiert einfach so lange bis er Glück hat.
05
Passwort-Analyse
Das Admin-Passwort konnte erraten werden. Welches dieser Passwörter wäre am schwersten zu knacken?
admin123 — kurz, häufig verwendet
Berlin2024! — Stadtname + Jahr + Sonderzeichen
P@ssw0rd — klassische Ersetzungen
Tisch!Mond42#Wolke — drei zufällige Wörter + Sonderzeichen + Zahl
Hinweis anzeigen
Ein gutes Passwort ist entweder kurz aber sehr komplex (4 Zeichenarten, 8+ Zeichen) — oder sehr lang (25+ Zeichen, 2 Zeichenarten). Wörterbucheinträge und Jahreszahlen sind bekannte Muster.
06
Privilege Escalation
Nach dem Login sieht der Angreifer im Log: PRIVILEGE ESCALATION — admin → root. Was bedeutet das?
Der Angreifer hat sein Passwort geändert.
Der Angreifer hat sich höhere Zugriffsrechte verschafft als er eigentlich hat.
Der Angreifer hat das System neu gestartet.
Der Angreifer hat Dateien verschlüsselt.
Hinweis anzeigen
„Privilege" = Recht/Berechtigung. „Escalation" = Eskalation, Steigerung. „root" ist der höchste Benutzer im System — darf alles.
07
Verhaltensregeln: PC-Raum
Der Angreifer konnte das Admin-Passwort möglicherweise im Schulgebäude abfangen. Welche Regel hätte das verhindern können?
Schüler dürfen den PC-Raum ohne Lehrkraft betreten.
Man darf das Passwort eines anderen benutzen wenn man das eigene vergessen hat.
Jeder benutzt ausschließlich sein eigenes Konto und Passwort — niemand gibt sein Passwort weiter.
Computer werden am Ende des Unterrichts nicht heruntergefahren.
Hinweis anzeigen
Die Verhaltensregeln im PC-Raum sind nicht willkürlich — sie schützen das System. Welche Regel verhindert direkt, dass jemand fremde Zugangsdaten benutzt?
08
Zwei-Faktor-Authentifizierung (2FA)
Hätte eine Zwei-Faktor-Authentifizierung den Angriff verhindert? Was ist das Prinzip von 2FA?
Man braucht zwei verschiedene Passwörter nacheinander.
Man braucht neben dem Passwort einen zweiten Beweis — z.B. einen Code aufs Handy.
Das Passwort wird doppelt verschlüsselt gespeichert.
Man muss sich von zwei verschiedenen Geräten gleichzeitig einloggen.
Hinweis anzeigen
2FA = zwei Faktoren. Faktor 1: etwas das du weißt (Passwort). Faktor 2: etwas das du hast (Handy) oder bist (Fingerabdruck). Beide müssen stimmen.
Block 01 abgeschlossen
Du hast alle Aufgaben gelöst. Der Zugang zur Fallakte wurde freigeschaltet. Speichere deinen Fortschritt bevor du weiter machst.
